# 建置手冊

> 依照這份文件的步驟，帶開發者（可能不是工程背景）一步步完成整套系統的建立。
> 每一步都要給「完整程式碼」讓開發者複製貼上，並用**非常詳細、逐步驟**的文字說明操作方式
> （例如「點哪個按鈕」「哪個選單」），不要假設開發者懂 Apps Script 編輯器怎麼操作。

## 整體架構

```
Google 表單（導師填寫初始通報）
    ↓（自動連結）
Google 試算表：彙整總表（正本資料）
    ↓（Apps Script 自動分流/排序/去識別化）
├─ 各年級分流工作表（檢視表，供各年級老師查看）
└─ 去識別化總覽（檢視表，供首頁動態表格讀取）

Apps Script Web App（doGet 依 ?page= 參數路由）
├─ 無參數 → 首頁（動態表格 + 搜尋篩選）
├─ ?page=<角色代號> → 各角色通報頁（密碼登入 + 搜尋 + 更新）
└─ ?page=admin → 管理者後台（密碼登入 + 新增/修改/刪除任何案件）

GitHub Pages（<帳號>.github.io，純外殼）
└─ 導覽列 + iframe 嵌入 GAS Web App 網址
```

---

## Step 1：建立 Google 表單

1. 依照跟開發者確認過的表單規格，修改 `reference/Code.gs` 裡的 `createReportForm()` 函式
2. 開發者開啟要當資料庫的 Google 試算表 → 「擴充功能」→「Apps Script」
3. 全選刪除編輯器內容，貼上客製化後的 `Code.gs` 完整內容，`Ctrl+S` 儲存
4. 上方函式下拉選單選擇 `createReportForm`，點「執行」
   - 第一次執行會跳出 Google 授權要求：點「檢閱權限」→ 選帳號 →
     若出現「Google 尚未驗證這個應用程式」警告，點左下角「進階」→「前往...(不安全)」→「允許」
5. 執行完成後，點左側「執行紀錄」查看 Logger 印出的：
   - 表單網址（給導師填寫用）
   - 表單 ID（**請把這個 ID 貼到 `Code.gs` 裡的 `FORM_ID` 常數**）
6. **重要**：`createReportForm()` 只能執行一次，不要重複執行（會建立出重複的表單）

> 如果表單需要事後補加題目（例如漏了一題），可以另外寫一個一次性函式（參考
> `docs/踩坑記錄.md` 坑3），新增後記得同步更新 `HEADER_NAMES`、確認新欄位落點。

## Step 2：初始化試算表結構

1. 把貼有正確 `FORM_ID` 的 `Code.gs` 完整內容再貼一次到編輯器（確保 FORM_ID 是最新的）
2. 上方函式選單選 `setupSpreadsheet`，執行
3. 確認：
   - 表單回應工作表被自動改名為「彙整總表」，且補上系統欄位標題（案件編號/排序值/
     狀態/結案/最後更新時間/最後更新人員 等）
   - 自動建立各年級分流工作表 + 去識別化總覽
4. `setupSpreadsheet` 可安全重複執行（不會重複建立或搬動已存在的欄位/工作表）

## Step 3：安裝觸發器

1. 上方函式選單選 `installTriggers`，執行
2. 確認安裝了 3 個觸發器：表單送出時、手動編輯彙整總表時、試算表結構變更時
   （左側「觸發條件」圖示可以查看確認）

## Step 4：設定密碼

1. Apps Script 編輯器左側齒輪「專案設定」→「指令碼屬性」
2. 依照跟開發者確認的角色命名，新增對應的密碼屬性（例如 `PWD_TRIAGE`/`PWD_REST`/
   `PWD_HOSPITAL`/`PWD_ADMIN`），開發者自行輸入密碼值
3. AI 助理全程不需要、也不應該知道實際密碼內容

## Step 5：部署 GAS 網頁應用程式

1. 右上角「部署」→「新增部署作業」→ 類型選「網頁應用程式」
2. 執行身分選「我」，誰可以存取選「所有人」
3. 部署後取得網頁應用程式網址（`.../exec` 結尾）
4. 測試：直接開啟網址（首頁）、`網址?page=<角色代號>`（各角色登入頁）、
   `網址?page=admin`（管理者後台）

> 之後修改 `Code.gs` 內容，只要在編輯器貼上新版存檔即可生效，**不需要重新部署**
> （除非有建立「已命名版本」的部署）。

## Step 6：Logo 處理（若開發者有提供圖檔）

如果 logo 背景是純色（通常是白色），可以用 Python + Pillow 做去背 + 裁切 + 縮圖 + 轉 base64：

```python
from PIL import Image
img = Image.open(src).convert("RGBA")
datas = img.getdata()
newData = []
threshold = 235
for r, g, b, a in datas:
    if r >= threshold and g >= threshold and b >= threshold:
        newData.append((255, 255, 255, 0))
    else:
        newData.append((r, g, b, a))
img.putdata(newData)
img = img.crop(img.getbbox())  # 裁切多餘透明邊框

# 縮圖成 256x256、置中留白，適合放圓形容器
size = 256
canvas = Image.new("RGBA", (size, size), (0, 0, 0, 0))
ratio = min((size*0.86)/img.width, (size*0.86)/img.height)
resized = img.resize((int(img.width*ratio), int(img.height*ratio)))
canvas.paste(resized, ((size-resized.width)//2, (size-resized.height)//2), resized)
canvas.save("logo_首頁用_256.png")
```

再把產生的 PNG 轉 base64（`[System.Convert]::ToBase64String(bytes)` 或
Python `base64.b64encode`），貼到 `Code.gs` 的 `SCHOOL_LOGO_BASE64` 常數裡。
若沒有提供 logo，`SCHOOL_LOGO_BASE64` 留空，首頁會自動顯示預設校徽 emoji。

## Step 7：建立 GitHub Pages 首頁外殼

1. 確認開發者的 GitHub 帳號、以及要用哪種網址形式：
   - `<帳號>.github.io`（帳號首頁形式）：repo 名稱必須完全等於這個字串、**必須是 public repo**
   - 或用某個 repo 的 `<帳號>.github.io/<repo名稱>` 形式（可以 private，但免費方案 Pages 只支援 public）
2. 建立該 repo（若用帳號首頁形式且尚未存在）
3. 以 `reference/github-pages-index.html` 為基礎，替換：
   - 標題、校名
   - 導覽列連結（表單網址、各角色 `?page=` 網址）
   - 首頁 iframe src（GAS Web App 網址，不加任何 `?page=` 參數）
4. Push 到該 repo 的 `main` 分支，`index.html` 放在根目錄
5. 等待約 30 秒~1 分鐘讓 GitHub Pages 建置完成，訪問確認

## Step 8：整合測試

依序測試：
1. 用測試帳號填寫表單，確認彙整總表自動產生案件編號/排序值/狀態
2. 各年級分流表、去識別化總覽是否正確同步
3. 各角色通報頁：密碼登入 → 搜尋 → 更新 → 確認彙整總表同步更新
4. 管理者後台：新增/修改/刪除案件，確認首頁動態表格同步
5. 手機版測試：務必確認每個 `doGet` 分支都有加 `viewport` meta 標籤（見踩坑記錄坑8）
6. 用瀏覽器自動化工具（如 Playwright）對正式網站做一輪基本功能驗證

## Step 9：正式上線前

1. 演習/正式使用前一天：透過管理者後台清空測試資料
2. 密碼交付給對應角色的負責人員
3. 確認演習/正式使用當天的操作 SOP
